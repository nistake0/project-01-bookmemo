### 3. 本番ビルドでのデバッグログ抑制

- **不具合修正**: `devLog` / `logger` において、ブラウザで `process` 未定義のときに `getAppEnv` が常に `development` になる問題を修正
- **対応**: `process.env.*` を直接参照し、Vite の `define` と連携
- **vite.config**: `loadEnv` + `define` で `NODE_ENV` / `VITE_LOG_LEVEL` / `VITE_DEBUG_LOGS` を埋め込み
- **コード置換**: 主要な `console.log` を `devLog` に置換、logger テスト拡張
- **bug-feature-memo**: 完了タスクを反映

### 6. リーディングカード xlsx インポートパイプライン

Excel の読書管理シートからアプリ向けバンドル（JSON）を生成するオフライン用スクリプトを `scripts/import-reading-card/` に追加し、設計・運用メモを `doc/import-reading-card-tooling.md` に整理してコミットした（`ad60548`）。

- **フロー**: `preview.mjs`（環境読み込み・バンドル出力）、`parseXlsx.mjs`（シート解析）、`buildBundle.mjs`（書籍グループ・Google 照会・レポート）、`apply.mjs`（適用）、スタブ向け `enrichStubBooks.mjs`
- **評価**: `rate` 列のみ（1〜5、空・不正は 0＝未評価）。旧 xlsx 色情報・`jszip` 依存は削除
- **メモ「名前」**: （再読）除去、改行 or 半角 `-`（前後空白）で書籍名 / `storyTitle`、末尾全角1桁は `volumeLabel` と検索用タイトル。`__orphan__` は正規化書名単位で名寄せ、`sources` は行単位で蓄積
- **年シート**: 検索用に名前の改行を折り畳み。`きっかけ` 列があれば引用風メモ＋タグ `きっかけ`
- **Google Books**: 特定タイトル向け ISBN 決め打ち、書籍が確定しない場合は **stderr** で検知しやすくした。スタブは `npm run import:enrich-stubs` で再検索し `resolved` / `googleBooks` を更新
- **その他**: `.gitignore` に `.google-books-api-cache.json`（成功レスポンスキャッシュ予定・未実装）を追加。`scripts/import-reading-card/README.md` からツール全体 doc へリンク
