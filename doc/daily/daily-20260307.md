### 2. セキュリティ対策（優先度：高）

#### 2.1 APIキー露出対応

- **doc/daily/daily-20250928.md**: 記載されていたGoogle Books APIキーを削除し、プレースホルダーに置換
- **.github/workflows/deploy.yml**: `.env.production`の内容や`grep`結果をログ出力する処理を削除（機密漏洩リスク回避）
- **doc/production-deployment-guide.md**: 「APIキー露出時の対応手順」を追加（無効化、再発行、GitHub Secrets更新の手順）

#### 2.2 APIキー再発行（ユーザー実施）

- Google Cloud Console で新規APIキー発行
- アプリケーション制限: ウェブサイト（開発・本番URLを個別指定）
- API制限: Books API のみ
- GitHub Secrets `VITE_GOOGLE_BOOKS_API_KEY` を更新
- `.env.local` の `VITE_GOOGLE_BOOKS_API_KEY` を更新

### 3. セキュリティ対策（優先度：中）

#### 3.1 CSP（Content Security Policy）の導入

**index.html** に以下を追加：

- `Content-Security-Policy` メタタグ
  - default-src, script-src, connect-src, img-src, style-src, font-src, frame-src
  - object-src 'none', base-uri 'self', form-action 'self', frame-ancestors 'self'
- `X-Content-Type-Options: nosniff`

**対策理由**: XSS軽減、クリックジャッキング防止、MIMEスニッフィング防止

#### 3.2 本番環境でのデバッグ機能無効化

**src/utils/errorLogger.js**:

- `registerDebugCommands()` を `PATHS.IS_DEVELOPMENT()` 内でのみ実行するよう変更
- `bookmemoDebug.getLocalStorage()` 等が本番で利用できなくなり、localStorageの機密情報露出を防止
- `beforeunload` 内の `console.log` を開発時のみに限定

### 4. セキュリティ対策（優先度：低）

#### 4.1 Firestoreルールの簡素化

**firestore.rules** - `statusHistory` サブコレクション：

- 冗長な `allow create` を削除（`allow read, write` に create が含まれるため）
- `isAuthenticated()` ヘルパーで統一

### 5. CSP修正（openBD API）

ISBN書籍検索で以下のエラーが発生：

```
Connecting to 'https://api.openbd.jp/...' violates the following Content Security Policy directive: "connect-src ..."
```

**対応**: `connect-src` に `https://api.openbd.jp` を追加。openBDは公開APIであり、セキュリティリスクは低い。
